## Feedback Collection

### Feedback Form

**Categories:**
1. **Bug Reports**
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots/videos
   - Device/browser info

2. **Feature Requests**
   - Description
   - Use case
   - Priority (Nice to have / Important / Critical)

3. **UX Feedback**
   - Confusing elements
   - Suggested improvements
   - Flow issues

4. **Performance**
   - Loading times
   - Responsiveness
   - Battery usage (mobile)

5. **Content**
   - Chapter clarity
   - Audio quality
   - Canticle effectiveness

## Testing Guidelines

### Do's ‚úÖ
- Test on your preferred device first
- Try to maintain daily streak
- Report bugs immediately
- Share honest feedback
- Ask questions in Slack
- Suggest improvements

### Don'ts ‚ùå
- Share screenshots publicly
- Discuss features before launch
- Use multiple accounts
- Share login credentials
- Submit fake feedback

## Beta Testing Agreement

All testers must agree to:
1. Maintain confidentiality until launch
2. Provide honest, constructive feedback
3. Complete minimum testing tasks
4. Report bugs promptly
5. Participate in check-in calls (optional)
6. Allow use of anonymized feedback
